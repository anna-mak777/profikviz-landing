# Перенос лендинга в Тильду

Этот лендинг встраивается **как один HTML-блок** на страницу Тильды (например, на `proftreker.ru/profiquiz`). Тильдовские блоки шапки/футера и форма заявки добавляются отдельно, в админке. Цель — чтобы при переносе ничего не «слетело» из стилей, картинок и шрифтов.

---

## Что должно быть в Тильде до встраивания

1. **Страница** в Тильде с правильным URL (например, `/profiquiz`).
2. **Блок шапки сайта** ПРОФТРЕКЕР сверху (если он общий для всего сайта).
3. **Блок формы заявки** под HTML-блоком (см. ниже про якорь `#register`).
4. **Блок футера** снизу (если общий).
5. Папка «Файлы» с залитыми PDF трёх документов (получим CDN-ссылки).

---

## Шаг 1. Заливаем картинки в Тильду

Тильда не отдаст картинки из нашей папки `images/` напрямую — нужно положить их на её CDN.

**Как:** в админке Тильды → «Файлы» (на уровне сайта) → загружаем все файлы из `images/`, **сохраняя структуру подпапок** (или без — Тильда плоская, тогда нужно будет переименовать пути в коде).

После загрузки каждый файл получает CDN-ссылку вида:
```
https://static.tildacdn.com/tild3661-3133-4337-a232-633538663239/Logo_Evraz.svg
```

**Список файлов, которые точно надо залить** (то, что упоминается в коде):
- `Logo_Proftreker_dark.svg` (шапка, футер)
- `kot_yellou.svg` (hero)
- `Logo_Evraz.svg`, `Logo_Eurohim.svg`, `Logo_FosArgro.svg`, `Logo_UPK.svg` (hero логотипы)
- `cards/photo_2024-08-01_16-46-03.jpg` (о проекте)
- `Komanda_Proftreker.jpg`, `dety3.jpg`, `dety5.jpg` (новости)
- `deti1.jpg`, `dety6.jpg`, `dety5.jpg`, `zal1.jpg` (раунды)
- `suvenirs/Penal_orange.JPG` (подарки)
- `dety4.jpg` (площадка)
- `cards/photo_2024-08-01_15-48-36.jpg` (каталог профессий)
- `fest1.jpg` (фестиваль)

**Совет:** удобнее всего собрать копию папки `images/` без неиспользуемых файлов и зипануть для Тильды.

## Шаг 2. Меняем пути к картинкам в `index.html`

В коде везде стоит `src="images/..."`. После загрузки в Тильду — заменяем на CDN-ссылки.

Самый быстрый путь — **скрипт замены** (готова сделать, если дашь маппинг файлов).

Альтернатива — оставить относительные пути и захостить папку `images/` рядом с сайтом (например, на subdomain). Но это лишний хостинг, лучше через Тильду.

---

## Шаг 3. Вставляем HTML-блок

1. В админке Тильды на нужной странице → добавить блок типа **«T123» — «HTML-код»** (или «Свой HTML»).
2. Скопировать содержимое `<body>...</body>` нашего `index.html` (БЕЗ тегов `<html>`, `<head>`, `<body>`).
3. Скопировать содержимое `<style>` из `css/style.css` и обернуть в `<style>...</style>` **внутри** HTML-блока.
4. Скопировать `js/main.js` и обернуть в `<script>...</script>` **внутри** HTML-блока в конце.

**Альтернатива по CSS/JS:**
- В админке Тильды → «Настройки сайта» → «Ещё» → «HTML-код в HEAD» или «BODY end».
- Можно положить туда `<link rel="stylesheet" href="https://CDN/style.css">` и `<script src="https://CDN/main.js">`, если css/js залит на тот же тильдовский CDN.

**Про префикс `pq-` в классах:** он специально сделан, чтобы наши стили не конфликтовали с тильдовскими. То есть встраивание не сломает существующий дизайн Тильды.

---

## Шаг 4. Шапка и футер в Тильде

В нашем коде есть `<header class="pq-header">` и `<footer class="pq-section pq-footer">` — это **дубликаты** того, что обычно делает Тильда отдельными блоками.

**Решение 1 (рекомендую):** удалить нашу шапку и футер из HTML-блока, использовать тильдовские. Так шапка/футер будут одинаковыми на всех страницах сайта.

**Решение 2:** оставить наши шапку и футер, не использовать тильдовские. Подходит, если у проекта свой уникальный дизайн шапки.

Если выбираем решение 1 — нужно **удалить из HTML-блока**:
- весь тег `<header class="pq-header">...</header>`
- весь тег `<footer class="pq-section pq-footer">...</footer>`
- бургер-JS станет неактуальным (можно оставить — он просто ничего не найдёт и тихо пройдёт мимо)

## Шаг 5. Форма заявки

В коде кнопка «Зарегистрироваться» ведёт на `href="#register"`. Якорь `id="register"` сейчас на блоке финальной CTA (тёмно-фиолетовый блок «Зарезервируйте игру»).

**Что делаем в Тильде:**
1. Добавляем тильдовский блок формы (T754 «Форма с кнопкой» или T830 «Многошаговая форма») **сразу после нашего HTML-блока**.
2. В админке этого блока в «Настройках блока» → «Web» → задаём id `register`.
3. Тогда клик по любой кнопке «Зарегистрироваться» в нашем HTML прокручивает страницу к форме.
4. **Удаляем нашу секцию финальной CTA из HTML-блока** (тёмно-фиолетовый блок с заголовком «Зарезервируйте игру для своей школы») — её роль теперь играет тильдовский блок с формой.

Поля формы (рекомендация):
- ФИО
- Организация (школа / колледж / лагерь)
- Город
- Телефон
- E-mail
- Какая серия игр (Летняя / Осенняя) — radio
- Желаемая дата — текст или дата
- Комментарий — текст
- Согласие на обработку ПД — чекбокс

---

## Шаг 6. Head: favicon, OG-теги

Наш блок `<head>` в Тильду не попадёт — Тильда генерирует head сама. То что мы прописали (favicon, OG, meta description) **нужно перенести в админку Тильды**:

1. **Настройки страницы** → «SEO» — Title, Description.
2. **Настройки страницы** → «Facebook и SEO» — загружаем `og-image.jpg`, прописываем og:title, og:description.
3. **Настройки сайта** → «Главное» — favicon (загружаем `favicon.svg` или конвертируем в PNG).
4. Apple-touch-icon — туда же.

## Возврат к разработке после встраивания

Если после деплоя нужно что-то поправить:
1. Правим локальный `index.html` / `css/style.css` / `js/main.js`
2. Тестируем локально (открыть `index.html` в браузере)
3. Коммитим в git
4. Копируем обновлённый код в HTML-блок Тильды (только то, что внутри `<body>` — без шапки и футера, если их перенесли в тильдовские блоки)
