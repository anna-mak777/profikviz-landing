## CTA-стратегия (из wireframe)

Одна формулировка повторяется **3 раза**: header (secondary, контурная) → hero (primary, заполненная) → финал (primary, заполненная). Кнопка ведёт на **живую форму Тильды** (вписывается в боевую сборку, не на этапе разработки).

## Дизайн-приёмы из wireframe

- Заголовки секций — метка `━━` маленькими буквами в моноширинном шрифте, потом крупный H2
- Иконки — квадраты со штрихом, без обводки-круга (стиль «технических чертежей»)
- Места под фото — пунктирные рамки с подписями типа `[ ФОТО · ЗАЛ С ИГРОЙ ]`
- Разделители — тонкие линии `#E5D9C5` / `#D4C5AC` между блоками; толстые `#31263B` 1.5px внутри секций (рамки карточек, таблицы)
- Кнопки прямоугольные с малым радиусом (4–10px)
- В hero и финальной CTA внутри кнопки — стрелка `→` цвета `#FF3770`

## Шрифты

- Manrope (заголовки 700/800, тело 400/500) — основной
- В wireframe для аннотаций используется JetBrains Mono — в боевом лендинге **тоже подключаем** для меток-уголков (`━━ О ПРОЕКТЕ`, технических лейблов вроде `РАУНД 01`, дат `ноябрь 2026`). Это визуальный приём wireframe, который должен остаться.

Подключение:
```html
<link href="https://fonts.googleapis.com/css2?family=Manrope:wght@400;500;600;700;800&family=JetBrains+Mono:wght@400;500;600&display=swap" rel="stylesheet">
```
